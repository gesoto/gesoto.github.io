A design system supporting a cloud platform in the building construction process (Planning, Provisioning, Installation, Management, Maintenance, etc).

I managed, designed, and developed a library of UI components that were used by both design and development to mock-up, prototype, and develop the final product.

Concept

Katerra's technology-focused operations made use of multiple in-house software solutions that required design systems and specialized components to improve efficiency and usage experience.

Cloud-based unified platform supporting Katerra's advanced building ecosystem and lifecycle.

Planning

Architecture information, story maps, and user analysis, helped understand and assess the type of UI Controls and Components required to assemble a Design System.

Splitting features into distinct module components made it easier to swap, add, or remove them as the platform adapted to future requirements.

Exploration & Prototyping

Exploration, prototyping and iteration were key while designing custom components.

High-Fidelity Prototype & Layout Samples

Together with a library of code components I compiled multiple examples into a single high-fidelity prototype, which by running in the target platform allowed developers to reference these patterns when building applications modules, often giving them a head start in the development process.

Component Library

All the components were categorized and tracked for status, progression, target, type, etc. This helped me name, manage, and build these components in an efficient manner.

Design & Development Components

I developed two libraries of UI components that shared a similar source and therefore able to keep visually in sync throughout design and development by avoiding common hand-off issues.

Design components could easily be imported into UI design tools like Framer to quickly mockup and test patterns.

Development components were showcased through Storybook, a platform to develop and share UI components in isolation.
